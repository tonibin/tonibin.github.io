Introduction

Welcome to tetra, an interpreter for Forth-79. tetra (formerly known as qartus) is a complete console environment adhering to the Forth-79 Standard, with some additions, two editors (a line-editor and a screen-editor), a complete library and documentation.

(from the manual)
"Why Forth? Perhaps because I enjoy programming, I find Forth appealing, and I love creating something by assembling smaller component. But principally because Forth is the most fascinating imperative language ever created, in my opinion."

"Why Forth-79? I chose Forth-79 because it represents the first standardization of the Forth language, offering simplicity and historical significance, and it was the version I initially studied."

"I don't belong to any Forth circle, no Forth group, no Forth organization; no university, no school, no structure. And I'm not a professional programmer, so please don't tell me 'you should've done this that way or that this way!' because I know that my programming style is pretty naive. I want to make my algorithms work: speeding them up and making them beautiful are tasks that go beyond my interests."

Installation

The tetra executable, before use, must be compiled, installed and configured. I suggest the following very easy procedure in few steps (use the terminal):

1) Copy the package somewhere, and unpack it; it will unpack into a directory named, say, tetra.XXX.src (where 'XXX' is a string specifying the program version);

2) Get into tetra source directory, open the file custom.h and change what you think must be changed. The file has auto-explicative comments;

3) Then type the following (be sure to have gcc installed):
$ make
$ make install

The installation may require root privileges if you try to put tetra into system directories, as suggested by default in the Makefile. If you have administrative access, but you type from your own login, type

$ sudo make install

4) Then you can type the following to set up the environment
tetra --config

The --config installation (not mandatory) creates the hidden folder .tetra/ in your $HOME directory and the subfolder .tetra/blocks/; then all installation block files are copied therein. Installation blocks are named with the reserved number range 20000-29999; all other numbers are at your disposition. If you want to alter reserved number range blocks, remember to rename them with another number, because one next version can overwrite all installation blocks.

5) now type the following
tetra --version

to see the version number, or better

tetra --help

to see all basic options of tetra. Here it is! tetra is installed and can be run! Remember also to read the manual, to know the internal secrets of tetra!

What's new:
  • the name of the compiler is now tetra, because there already was a Quartus Forth compiler somewhere in the world... The name tetra was suggested by Bruce, and I like it.
  • many corrections were applied, both in the compiled words and in the library words. Some bugs were solved, and some new words added (mainly SLITERAL and ?DO)
  • some options were added, above all --pad-distance and --evaluate, and also --strict
  • the library has been enhanced with many more tools, and now it has more that 350 words; many new additions from the ANSI world and the Apple Forth-79;
  • see also the following CHANGELOG for detailed information about the changing from version Alpha to version Beta.

About tetra
  • improved the manual with additions and examples; new paragraphs and new topics about Forth were added; several nuisances and imprecisions were corrected;
  • corrected and updated the man page: there were too many errors in it!
  • corrected the makefile, that didn't include the pdf manual! What a shame! Besides, the make command didn't work as expected.
  • solved a bug for the usage of readline, that didn't manage well words beginning with the symbol '(', '\' and others that didn't work well; the change was made with the help of Chet Ramey, the current readline maintainer;
  • in typing commands, readline history now is updated only for non-repeating commands
  • added options --pad-distance= and --evaluate=""
  • added option -s/--strict and the variable STRICT-STATE; the strict state changes the functioning of WORD, the dot-quotes, the (..) comments and FORGET, to fully adhere to the Forth-79 Standard.
  • removed options -n, -r and -s (redundant) and added -S for statistics
  • added the colon-sys system, because the exploiting figure : foo [ : bar ; ] ; caused the system to be unresponsive.
  • solved a bug in DOES> that prevented the copying of the DOES> part in some cases, due to a wrong addressing of the variable currPC, which has now been removed and replaced with _PC
  • solved a bug in the (double) number input routine that didn't calculate correctly DPL in all cases
  • reviewed the vocabularies compilation, removed the VOC_MARK and voc_chain[]; now the chaining is done directly into the dictionary words; with this new protocol, there are no limits in the vocabularies creation, and this responds more strictly to the Forth-79 protocol.
  • added some more Forth files with interesting programming solutions.
  • compiled vocabularies with [COMPILE] use now the address link.
  • added controls to BASE, which cannot be altered beyond the 2..36 interval, or the system becomes unstable...
  • CREATE, CONSTANT, VARIABLE, 2CONSTANT, 2VARIABLE, FORGET and others now have an explicit run-time counterpart that is shown in compiled form
