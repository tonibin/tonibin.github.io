Introduction

Welcome to apple, the APL for the 1130 machine, adopting the 1969 manual as source. I reversed here all the experience I gained in studying and using this wonderful language, which is not a programming language in the common sense, but rather a mathematical language, and this has some meaning to me. You don'w need special keyboards or combination of characters, to use apple, because only regular ASCII characters are used. apple is simple and powerful, full of features and with a full documentation. It's a complete, maybe not so easy tool, but anyway useful for learning this wonderful mathematical language.

Installation from sources

The apple executable for the Linux and Unix environments, must be compiled and installed before usage. I suggest the following very easy procedure in 4 steps (use the terminal):

1) Copy the package somewhere, and unpack it; it will unpack into a directory named apple.XXX.src (where 'XXX' stands for the version string);

2) Get into apple source directory, open the file custom.h and change what you think must be changed. The file has auto-explicative comments;

3) Now type the following (be sure to have gcc installed):

$ make
$ make install

The latter may require root privileges if you try to install apple into system directories, as suggested by default in the Makefile. In this case, if you have the administrative access, as a common user, type

$ sudo make install

and the program will install.

4) now type the following to test the correct installation:

$ apple --version

to see the version number, or better

$ apple --help

to see all basic options of apple (type )HELP to get a complete list of commands, and )OFF to exit). Here it is! apple is installed and can be run! Remember also to read the manual, to know the internal secrets of the APL language!
